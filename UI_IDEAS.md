## 1. Наблюдения по нашим текущим экранам (self-review)

🔥 **Компас-циферблат на дашборде** — сильный якорь экрана, читается мгновенно. Оставить как «герой».
💡 **Два варианта дашборда** в мокапах: короткий (карточка «Активный переход» с 6 цифрами) и полный (большой компас + слайдеры XTE). Стоит решить: короткий = Today, полный = отдельный «режим перехода»/полноэкранный инструмент.
💡 **3 темы (Light Blue / Sea Green / Sunset Purple)** — хорошо для персонализации, но проверить читаемость Sunset Purple на солнце.
🧊 **Слайдеры «1.5 nm / 0.3 nm» под компасом** (на полном дашборде) — неочевидно, что это. Подписать или убрать.
🔥 **MOB как единственный красный** выдержан — не ломать это правило нигде.

## 2. Идеи из парусно-навигационных приложений

_(сложность: 🟢 просто · 🟡 средне · 🔴 сложно — относительно текущей кодовой базы)_

- [ ] 🔥🟢 **Трек, раскрашенный по скорости** (Navionics). Полилиния перехода —
  градиент от синего (дрейф) к циану (полный ход). У нас скорость уже лежит в
  каждой TrackPoint — это чисто рендер. Мгновенный «вау» на карте и в деталях рейса.
- [ ] 🔥🟡 **Авто-детект начала перехода** (Argo). Скорость >2 kn дольше 5 минут и
  запись не идёт → пуш «Похоже, вы вышли — начать запись?» (одной кнопкой из
  нотификации). Закрывает главный провал всех журналов: «забыл включить».
- [ ] 💡🟡 **Импорт маршрута GPX** (Aqua Map / iNavX). Пунктирный planned route уже
  рисуется — осталось распарсить `<rte>` из файла (экспорт УЖЕ есть, импорт — зеркало).
- [ ] 💡🟢 **Круг вытравленной цепи у якоря** (Aqua Map). Поле «вытравлено N м» +
  второй пунктирный круг поверх дрейф-круга: видно запас до алярм-радиуса.
- [ ] 🧊🔴 Погодная маршрутизация (PredictWind) — не наш вес, не сейчас.

## 3. Идеи из морских/яхтенных приложений

- [ ] 🔥🟢 **Барометр iPhone** (у всех конкурентов — только по подписке!). В телефоне
  есть датчик давления (CMAltimeter). Пишем давление в каждую запись лога
  автоматически + спарклайн тренда за 3 часа со стрелкой ↘︎/→/↗︎ на Today.
  Падение >2 гПа/3 ч — мягкое предупреждение. Офлайн, без API, честная морская фича.
- [ ] 🔥🟡 **Живая погода Open-Meteo Marine** — бесплатный API без ключа (ветер,
  волна, порывы по координате). Закрывает «Coming soon» в Weather честными данными;
  прогноз кэшируется на сутки → работает и без сети. (Обкатан в ISSA-тренажёре.)
- [ ] 🔥🟢 **Чек-листы отхода/прихода**. Шаблон pre-departure (топливо, кингстоны,
  прогноз, brief экипажа...) с галочками; завершение чек-листа пишется событием в
  судовой журнал. Просто Form + шаблоны, а выглядит как «серьёзный инструмент».
- [ ] 💡🟢 **Восход/закат/сумерки** для текущей позиции (алгоритм NOAA, офлайн).
  Плитка на Today: «закат 21:14 · через 2 ч 40 мин» — планирование прихода засветло.
- [ ] 💡🟡 **Вахтенный планировщик**: экипаж уже есть — раскидать вахты 3/3 или 4/4,
  локальные нотификации «через 15 мин ваша вахта», смена вахты — событие в лог.
- [ ] 🧊🟢 Карточка MMSI/позывного из Vessel на экране Safety (быстро продиктовать
  в рацию) + журнал радиосвязи.

## 4. Идеи из фитнес/трекинг-приложений (для статистики и истории)

- [ ] 🔥🟢 **Share-карточка перехода** (Strava). Красивая картинка: миникарта трека +
  дистанция/время/max скорость/парус-мотор — через SwiftUI `ImageRenderer`, шарится
  в одно касание. Бесплатный маркетинг в каждом яхтенном чате.
- [ ] 🔥🟢 **Пост-переходная сводка** (Apple Fitness «Workout complete»). После
  «Стоп» — полноэкранная карточка итогов с мягкой анимацией вместо простого
  возврата на Today. Уже есть все цифры, нужен один экран.
- [ ] 💡🟢 **Сезонная цель** (Fitness rings): «500 nm за сезон» — кольцо прогресса
  на Today/в виджете. Данные уже считаются для стрик-виджета.
- [ ] 💡🟡 **Фото в записи лога** (Komoot highlights). Разрешение Photos уже
  запрашивается для судна; добавить фото к LogEvent → маркеры-фото на треке.
- [ ] 🧊🟡 Годовой хитмап походов (Strava) — на потом, когда накопится история.

## 5. ТОП-идеи к решению

Сведение по критерию «эффект / трудозатраты» (🟢 = можно сделать за день-два):

| # | Идея | Слож. | Почему в топе |
|---|---|---|---|
| 1 | Авто-детект начала перехода | 🟡 | лечит «забыл включить» — главный churn журналов |
| 2 | Share-карточка перехода | 🟢 | виральность + повод открывать историю |
| 3 | Барометр + тренд | 🟢 | «настоящая морская» фича, у конкурентов платно, офлайн |
| 4 | Живая погода Open-Meteo | 🟡 | закрывает Coming soon бесплатным API |
| 5 | Чек-листы отхода | 🟢 | «серьёзный инструмент», пишется в журнал |
| 6 | Трек по скорости | 🟢 | самый дешёвый визуальный вау |
| 7 | Пост-переходная сводка | 🟢 | эмоциональная точка в конце каждого рейса |
| 8 | Восход/закат | 🟢 | планирование прихода, ноль зависимостей |
| 9 | GPX-импорт маршрута | 🟡 | замыкает цикл экспорт↔импорт |
| 10 | Сезонная цель-кольцо | 🟢 | мотивация возвращаться в сезон |

> ⚠️ Перед фичами — спринт 1 из `docs/AUDIT_2026-07-11.md` (тревога, фоновый
> фан-аут GPS, бюджет виджетов, перф трека): фундамент важнее украшений.
