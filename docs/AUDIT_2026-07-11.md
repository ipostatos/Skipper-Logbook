## P0 — критично (безопасность и ядро записи)

### 1. Раздача GPS-фиксов живёт во View-слое
`App/RootView.swift:52-58` — единственное место, где фиксы доходят до движков:
`.onChange(of: location.currentLocation)` → `recorder.ingest / anchorWatch.ingest /
mob.ingest`. Это модификатор SwiftUI-вью: если сцена выгружена/не обновляется в
фоне, **якорная тревога и MOB-дальность перестают получать координаты**, хотя
CLLocationManager продолжает слать фиксы. Тревога, зависящая от жизни UI-иерархии, —
это декорация.
**Фикс:** фан-аут в не-UI слое — координатор, подписанный прямо на
`LocationManager` (замыкание/делегат из `didUpdateLocations`), созданный в
`SkipperLogbookApp.init` рядом с движками.

### 2. Якорная тревога будит один раз — и её глушит беззвучный режим
`Core/Location/AnchorWatchEngine.swift:96-101, 134-143`:
- латч `alarmLatched` корректно давит дребезг, но пока лодку тащит, тревога
  **не повторяется** — один хапт + один звук + одна нотификация;
- у нотификации нет `interruptionLevel = .timeSensitive` → Фокус/беззвучный её
  съедают; `.critical` (с entitlement) не запрошен даже в роадмапе;
- `AudioServicesPlayAlertSound` не звучит, если телефон в беззвучном.
**Фикс:** повторять тревогу каждые 15–30 с, пока `isDragging` (и после возврата в
круг — «all clear»-нотификация); `interruptionLevel: .timeSensitive` немедленно;
подать заявку на critical alerts entitlement — это флагманская фича.

### 3. Виджеты обновляются «в лоб» на каждом фиксе — бюджет сгорит за час
`App/RootView.swift:57,72-109` + `Core/Widget/LiveActivityController.swift:18-23`:
на каждый фикс (фильтр 5 м → каждые ~2-5 с в движении) выполняется
`fetch(все Voyage)` + `fetch(все MaintenanceItem)` + `fetch(Vessel)` + расчёт
статистики месяца + `WidgetCenter.reloadAllTimelines()`. У WidgetKit бюджет
~40–70 перезагрузок/сутки: через час записи **виджеты замирают до завтра**;
попутно — постоянная работа на main-акторе.
**Фикс:** Live Activity можно обновлять каждый фикс (локально не лимитируется),
`reloadAllTimelines()` — только по событиям (старт/стоп, смена waypoint) и по
троттлу раз в 10–15 мин; тяжёлые fetch-и — кэшировать/считать по событиям, а не
на фикс.

### 4. Live Activity: сирота после перезапуска + вечно «свежие» данные
`Core/Widget/LiveActivityController.swift`:
- `activity` держится только в памяти; после force-quit/перезагрузки контроллер
  не делает reconciliation по `Activity<VoyageActivityAttributes>.activities` →
  старая активность **висит на локскрине до 8+ часов**, `endActivity()` её не
  видит, а новый старт плодит дубль (`:27-41`);
- `staleDate: nil` (`:35,45`) → при обрыве GPS/убийстве приложения на локскрине
  **замороженные скорость и курс без признака устаревания** — для навигационного
  приложения это опасное враньё;
- виджет-таймлайн пишет `updatedEpoch`, но никогда его не читает
  (`SkipperWidgets/WidgetSupport.swift:34-39`) → после kill виджет показывает
  «Recording» вечно.
**Фикс:** в `init` подобрать/закрыть осиротевшие активности; `staleDate = now+120с`
на каждом апдейте; в таймлайне сравнивать `updatedEpoch` с now и рисовать
состояние «данные устарели / запись прервана».

### 5. Квадратичная деградация записи трека
- `Shared/Models/Voyage.swift:88-90` — `orderedTrack` **сортирует весь массив при
  каждом обращении**;
- `Core/Location/VoyageRecorder.swift:67` зовёт его на каждом фиксе ради `.last`;
- Today/Map/Statistics/VoyageDetail зовут его на каждый рендер
  (`TodayView.swift:339`, `MapView.swift:25-27`, `StatisticsView.swift:46`);
- `VoyageRecorder.swift:84` — `context.save()` на каждый фикс.
Дневной переход = 10–20 тыс. точек: сортировки тысяч элементов на main-акторе
каждые 2–5 с + запись на диск каждые 5 м.
**Фикс:** кэшировать последнюю точку в рекордере (или fetch с `fetchLimit 1` по
timestamp desc); сохранять батчами (раз в 10–15 с или каждые N точек); для карты —
упрощение линии (Дуглас-Пекер / прореживание по расстоянию) и инкрементальное
добавление вместо пересортировки.

### 6. Фильтрации качества фикса нет вообще
`Core/Location/LocationManager.swift:113-123` — принимается любой фикс: без
проверки `horizontalAccuracy` (< 0 = невалидный; > 50–100 м = мусор в трек и
дистанцию), без отбраковки первого кэшированного (старый `timestamp`).
`speed = max(0, speed)` превращает «скорость неизвестна» (−1) в честный 0.0 kn.
**Фикс:** отбрасывать фиксы с `horizontalAccuracy <= 0 || > 100` (для дистанции —
строже, ~50 м), первые секунды после старта игнорировать stale-фиксы; скорость
хранить как Optional.

### 7. SafetyView недостижим
`Features/Safety/SafetyView.swift` — экран с большим MOB, карточкой последнего
MOB (живые дальность/пеленг) и входом в якорную вахту — **не вызывается ниоткуда**
(единственная ссылка — его собственный `#Preview`; в `AppRouter` нет `.safety`,
в More-гриде нет плитки — `MoreMenuView.swift:14-24`, при этом README и
док-комментарии экран рекламируют).
**Фикс:** плитка Safety в More (или собственный таб) + маршрут в `AppRoute`.

### 8. «Призрак» экипажа при отмене добавления
`Features/Crew/CrewView.swift:48-58` — `CrewMemberEditView(member: newMember())`
вызывается **внутри ViewBuilder шита**, а `newMember()` сразу `context.insert`.
Свайп-вниз (Cancel) оставляет пустого члена экипажа (autosave), а повторное
вычисление builder-а может насоздавать несколько.
**Фикс:** вставлять в контекст только по «Done» (или строить объект вне контекста /
в дочернем контексте).

### 9. Пять экранов «More» бесполезны для реального пользователя
`Features/More/ReferenceViews.swift` — Equipment / Service notes / Season log /
Deviation table **не имеют UI создания записей** (ни toolbar, ни шита — проверено),
а «+» в Maintenance — `.comingSoon()` (`MaintenanceView.swift:43-45`). Реальный
пользователь стартует с пустой базой (сид только под `--seed-demo`) → все пять
разделов навсегда пустые, README же перечисляет их как фичи.
**Фикс:** формы добавления (это те же простые Form-шиты, что уже есть у Crew) —
либо честные «Coming soon» на плитках More.

## P1 — важно (корректность данных и продукт)

### 10. Истинный и магнитный курсы смешаны
`LocationManager.effectiveHeading` (`:71-73,128`) возвращает истинный ЛИБО
магнитный без маркировки; `MOBActiveView.swift:74` вычитает из него **истинный**
пеленг → стрелка наведения врёт на величину склонения (на Балтике ~5–8°).
Нигде в UI нет пометки T/M.
**Фикс:** для относительного пеленга использовать один базис (истинный:
`trueHeading`, при недоступности — COG); подписывать °T в интерфейсе. У вас же
есть целый тренажёр T-V-M-D-C в соседнем проекте :)

### 11. Состояние двигателя двоится и не переживает перезапуск
`AppState.engineOn` (чипы) и `VoyageRecorder.engineOn` (мото-часы) — два
независимых флага; после перезапуска приложения при живой записи оба сбрасываются
в off, хотя последним событием был `engineOn` → следующий тап пишет второй
`engineOn`, `engineSeconds` копятся неверно. Плюс расхождение:
`TodayView.toggleEngine` (`:357-361`) выставляет `recorder.propulsion = .engine`,
а `QuickActionsSheet.swift:66-69` — нет → разбивка Engine/Sails в статистике
зависит от того, откуда тапнули.
**Фикс:** единственный источник — рекордер; состояние восстанавливать из последнего
LogEvent при `fetchRecording`; квик-экшены и Today — через один и тот же метод.

### 12. Удаления данных нет, а когда появится — утекут аудиофайлы
Единственный `context.delete` во всём приложении — экипаж (без подтверждения,
`CrewView.swift:99-101`). Рейсы, судно, записи лога, голосовые заметки удалить
нельзя. При этом cascade на `voiceNotes` удалит строки, но `.m4a` в
`Documents/VoiceNotes` не удаляет никто (`removeItem` в проекте отсутствует).
**Фикс:** swipe-to-delete + confirmationDialog для рейса/записей/заметок; при
удалении VoiceNote — удалять файл; чистка осиротевших файлов при старте.

### 13. Запись голосовой заметки умрёт при блокировке экрана
`Info.plist:59-62` — `UIBackgroundModes = [location]`, режима `audio` нет.
Заметка «с руля», начатая и убранный в карман телефон = обрыв записи.
**Фикс:** добавить `audio` в UIBackgroundModes (осознанно, с обоснованием для
ревью) либо явно показывать «запись остановится при блокировке».

### 14. Готовность к App Store — блокеры
- **Нет `PrivacyInfo.xcprivacy`** (обязателен с мая 2024): используются
  required-reason API — UserDefaults (`VoyageSnapshot.swift:64-66` и др.),
  file-timestamp. Без манифеста — reject на аплоаде.
- **Permission-строки только на английском** (`Info.plist:51-58`), файла
  `InfoPlist.xcstrings` нет — RU-пользователь видит EN-промпты. Заодно в тексте
  Always-разрешения стоит упомянуть якорную тревогу (помогает ревью).
- **Виджеты и Live Activity целиком EN**: каталог не подключён к таргету
  (`project.yml:46-59`, `SWIFT_EMIT_LOC_STRINGS: NO`), все строки — литералы.
- ETA в виджете/LA — рукописный `"HH:mm"` (`ActiveVoyageWidget.swift:152-156`,
  `VoyageLiveActivity.swift:124-127`) — игнорирует 12/24-часовой формат юзера.

### 15. Миграции схемы не подготовлены
`PersistenceController.swift` — плоская `Schema([...])` без
`VersionedSchema`/`SchemaMigrationPlan`, а `makeContainer()` делает `fatalError`
→ первое же неудачное авто-мигрирование после релиза = краш на старте без
восстановления.
**Фикс:** ввести `SchemaV1` + план миграций до первого релиза; вместо fatalError —
попытка восстановления (бэкап стора + пересоздание с сообщением пользователю).

### 16. CI обманывает сам себя
`.github/workflows/ci.yml`:
- `| xcbeautify` — **xcbeautify не установлен** (ставится только xcodegen);
  с `set -o pipefail` пайп падает и `||`-фоллбек **гоняет clean build test второй
  раз** — каждый прогон CI делает двойную работу (`:40-54`);
- `swiftlint --strict ... || swiftlint` — строгий линт **никогда не завалит билд**
  (`:65`);
- симулятор не запинен по OS (`name=iPhone 16` без `OS=`) — недетерминизм между
  обновлениями образа; кэша brew/DerivedData нет; lint на macos-14, сборка на
  macos-15.
**Фикс:** `brew install xcbeautify` (или убрать пайп), убрать `|| swiftlint`,
запинить OS, добавить кэш.

## P2 — среднее (UX, локализация, полировка)

- **Локализационные дыры в UI при 100% каталоге**: значения чипов "ON/OFF",
  "UP/DOWN", "Add" (`StatusChip.swift:73-88`); a11y-лейблы "Man overboard" ×3
  (`TodayView:280`, `MapView:168`, `SafetyView:157`), "Quick actions"
  (`CustomTabBar:70`), "Beta version" (`Badges:16`); "kn max" и оси графика
  Time/Speed/Distance (`StatisticsView.swift:74-96`); «E/W» девиации
  (`ReferenceViews.swift:192`); `Двигатель ON/OFF` в RU-строках каталога.
- **Форматирование чисел не локализовано**: `String(format: "%.1f")`
  (`Double+Units.swift`) — всегда точка; RU ждёт запятую.
- **Переключатель единиц — заглушка по факту**: `AppState.unitSystem` никто не
  читает, все "kn/nm/m" захардкожены (`Units`), UI-пикер выключен. Честно, но
  тогда `UnitSystem` — мёртвый код: либо реализовать, либо выпилить до релиза.
- **Карта**: кнопки слоёв/центрирования/waypoint без accessibilityLabel
  (`MapView.swift:132-137`).
- **Today**: спарклайн «to waypoint» кормится `speedSamples.reversed()` —
  перевёрнутая скорость вместо дистанции (`TodayView.swift:153-154`); elapsed/ETA
  замирают без новых фиксов (нет тикера).
- **Statistics**: показывает только активный/первый рейс — статистика прошлых
  переходов недостижима (`StatisticsView.swift:14`); легенда пончика показывает
  нулевые доли «0.0nm (0%)».
- **NewVoyageSheet**: введённое имя пункта назначения сохраняется без координат →
  ETA «—» до ручного waypoint на карте (`:44-46`).
- **VesselEdit**: судно создаётся и сохраняется ДО редактирования, Cancel нет —
  свайп оставляет «New Boat» (`VesselView.swift:102-107`).
- **AudioLog**: у каждой строки свой плеер — предыдущий не останавливается при
  запуске следующего (`AudioLogView.swift:196`); `levels` растёт без предела при
  длинной записи (`AudioRecorderController.swift:118-128`).
- **Экспорт**: CSV без UTF-8 BOM (Excel покажет кракозябры в RU-заметках); GPX без
  `<extensions>` (скорость/курс точек теряются); `try?` на экспорте — молчаливый
  провал без алерта (`VoyageDetailView.swift:41-46`); tmp/Exports не чистится.
- **CoordinateFormatting**: краевой случай `59°59.9999'` → «59°60.000′»
  (минуты не переносятся в градусы; в DMS аналогично для 60′).
- **Компоненты**: `Avatar` декодирует UIImage в body без кэша; `DateFormatter`
  создаётся на каждый вызов (`Date+Formatting.swift`); `WaveformView` —
  `ForEach(0..<count)` антипаттерн; серифный ворднарк `AppHeader` противоречит
  собственной типографике (SF Pro only); размеры/отступы часто мимо токенов.
- **AnchorWatchEngine.updateRadius** — мёртвый код: радиус нельзя изменить после
  постановки (UI нет), хотя README обещает.
- **Тесты**: слой виджетов/LA не покрыт вообще; нет тестов metric/imperial;
  экспорт — только happy-path (нет заметки с переводом строки, пустого рейса);
  UI-скриншоты покрывают 8 экранов из ~19 и живут на `usleep`.

**Спринт 3 — «к релизу»:** пп. 13–16 + P2-локализация (privacy manifest,
InfoPlist.xcstrings, локализация виджетов, миграции, CI, единицы).
